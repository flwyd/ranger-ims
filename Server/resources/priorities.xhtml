IMS Priorities

The Ranger IMS associates a priority level with each incident. This can provide a way to identify incidents that need greater attention or can be safely ignored for a while, which may be useful for managing Operator and Khaki workload when there are many open incidents to manage.

In order for priorities to be useful, however, it's important that they be used consistantly by all users. To that end, the list below itemizes the priority levels and how they should be utilized:

  1. Emergency: we need to deal with this ASAP
  2. Urgent: we need to deal with this in a timely fashion
  3. Normal
  4. Not urgent: deal with this when resources are available
  5. Low priority: needs to stay open, but not actively being worked on
